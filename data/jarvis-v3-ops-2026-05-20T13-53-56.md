## Architecture Overview

JARVIS v3 replaces the single-shot Haiku responder with a full Sonnet agent loop capable of multi-turn tool chaining, self-modification with safeguards, and dynamic UI panel generation.

### Tool Tiers

**Tier 1 — Read (always free):**
- `get_channel_stats(channel)` — live YouTube data
- `get_recent_videos(channel, count)` — video performance
- `get_scheduled_queue(channel, days?)` — upload schedule
- `get_render_queue()` — active/pending renders
- `get_system_status()` — CPU/GPU/RAM/services
- `get_library_stats()` — image + keyword counts
- `get_local_videos()` — rendered files not yet uploaded
- `search_knowledge(query, scope?)` — file-walk grep across codebase
- `read_file(path)` — first 200 lines of any project file
- `run_thumbnail_test(topic, channel)` — spawn thumbnail-tester, score hook

**Tier 2 — Safe Write (immediate, no confirm):**
- `queue_video(topic, channel)` — append to queue.json
- `remember_preference(key, value)` — write jarvis-preferences.json
- `set_render_priority(videoId, priority)` — reorder queue
- `cancel_render(videoId)` — remove from queue
- `update_video_title(videoId, title)` — YouTube title update (staged)
- `update_video_description(videoId, description)` — staged
- `update_video_tags(videoId, tags)` — staged
- `add_to_playlist(videoId, playlistId)` — staged
- `schedule_upload(videoId, datetime)` — set publish time
- `flag_for_review(videoId, reason)` — internal QA flag
- `preview_codebase_change(filePath, instruction)` — Sonnet-generated diff (preview only)
- `render_custom_panel(title, html, css)` — dynamic UI panel

**Tier 3 — Powerful (verbal confirm required):**
- `run_overnight_batch(channel, videoCount?)` — detached batch process
- `apply_pending_modification(confirmed)` — write previewed file change
- `delete_video(videoId, reason)` — YouTube delete (1 at a time)
- `publish_video(videoId)` — set private → public
- `unpublish_video(videoId)` — set public → private
- `update_channel_description(channel, description)` — channel-level change
- `bulk_update_tags(channel, tagSet)` — mass tag update

**Hardcoded Protected (cannot be bypassed):**
- `delete_channel`
- `revoke_oauth_token`
- `delete_youtube_token_file`
- `bulk_delete_videos` (>3 at once)

## Confirmation Protocol

JARVIS calls `request_confirmation(tool, args, description)` before any Tier 3 action. This:
1. Stores `_pendingConfirmation` in memory
2. Returns a `confirm_pending:true` marker to the server
3. Server generates an amber confirmation panel
4. On next user message: `isConfirmation()` or `isCancellation()` intercepts before the normal agent loop
5. If confirmed: execute tool, call Sonnet for completion message
6. If cancelled: clear state, return "Standing by"
7. If new command: discard pending, start fresh

## Dynamic Panel System

- **`{type:"custom", title, html, css}`** — Sonnet generates arbitrary HTML/CSS
- `sanitizePanelHtml()` — DOMParser allowlist; strips `on*` attrs, validates hrefs
- `scopePanelCss()` — prefixes every CSS rule selector with a unique `.scope-{id}` class
- CSS variables available to Sonnet: `--cyan: #00E5FF`, `--amber: #FFB300`, `--bg-card: rgba(0,18,42,0.9)`, `--font-mono: 'Share Tech Mono', monospace`
