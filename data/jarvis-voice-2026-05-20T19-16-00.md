## Files Changed

### backend (jarvis/server.js)
- `synthesize()` — now calls `synthesizePiper()` first, falls back to `synthesizeEdge()`
- `synthesizePiper()` — spawns `python -m piper` with JARVIS model, outputs WAV
- `synthesizeEdge()` — edge-tts en-GB-RyanNeural, outputs MP3
- `JARVIS_CMD_SYS` — new system prompt returning `{spoken, actions}` JSON
- `POST /api/jarvis/command` — intent parsing via Haiku, returns spoken + actions
- `POST /api/jarvis/speak` — updated to handle WAV and MP3 content types

### frontend (jarvis/public/)
- `index.html` — MIC button added to command box
- `jarvis.css` — MIC button styles (.listening state = amber blink)
- `jarvis.js` — major additions:
  - `playJarvisVoice(text)` — fetches WAV from /speak, plays via Web Audio API
  - Audio-reactive r1 ring — scales with voice amplitude during speech
  - `handleActions(actions)` — open_channel, close_panel, highlight_metric
  - `sendCommand()` — now async, hits /api/jarvis/command, plays voice + executes actions
  - `toggleMic()` — Web Speech API mic input, auto-submits on result

## Flow
1. User types or speaks command
2. `sendCommand()` → POST /api/jarvis/command → Haiku parses intent → {spoken, actions}
3. Response shown in panel immediately
4. `playJarvisVoice(spoken)` → POST /api/jarvis/speak → Piper WAV → Web Audio API
5. Inner r1 ring pulses with audio amplitude while JARVIS speaks
6. `handleActions(actions)` executes: open channel overlay, highlight metrics, etc.
